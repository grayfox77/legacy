; CPU/LCD Timing
;
; These settings control the number of CPU cycles to execute per line for
; each LCD mode. Lowering them may increase speed, but might break games. On
; the other hand, setting them too high wastes valuable CPU time.
;
; A little explanation on the real hardware:
; The Gameboy executes 114 cycles during display of a single line. For the
; first 144 lines, the LCD runs through 3 modes: Horizontal Blanking, OAM
; Access and VRAM Access. Horizontal Blanking takes 51 cycles, OAM Access
; takes 20 cycles and VRAM Access takes 43 cycles. These numbers will change
; depending on the number of sprites on that line, but will always add up to
; 114.
;
; In GEM, OAM and VRAM Access are combined into a single 'mode'. This helps
; speed a bit and shouldn't break any games.
;
; Caution! Never set a value lower than 1 + Interrupt Latency (see below)
; Range: 1-255

Horizontal Blanking =           ; defaults to 51
OAM/VRAM Access     =           ; defaults to 63
Vertical Blanking   =           ; defaults to 114

; Interrupt Latency
;
; Controls the number of cycles it takes to enter an interrupt.
; Range: 0-9

Interrupt Latency   =           ; defaults to 5
